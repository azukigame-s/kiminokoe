1. **シナリオの内容を変更しない** — `scenarios/kiminokoe.md` の内容が正。JSONはこれを忠実に変換したものでなければならない

## ブランチ戦略

### 基本方針：trunk + 永続 release ブランチ + タグ

| ブランチ | 役割 |
|---------|------|
| `main` | 開発の統合ブランチ（常に最新） |
| `release/demo` | 体験版の配布用永続ブランチ（ここからビルド） |
| `fix/〇〇` | 不具合修正（main から分岐 → main マージ → release/demo に cherry-pick） |
| `feature/〇〇` | 機能追加・シナリオ実装（main から分岐） |

### ブランチ命名規則

- 区切り文字は `-`（ハイフン）。`_` は使わない
- `fix/` は Issue 番号を先頭に付ける（例: `fix/1-default-window-size`）
- `feature/` は機能名を簡潔に英語で（例: `feature/osekkai-scenario`）

### バージョン管理

- タグ名は `v{major}.{minor}.{patch}`（例: `v0.5.1`）
- バグ修正 → patch アップ、機能追加 → minor アップ
- 配布物は必ず `release/demo` のタグからビルドする

## ゲームの核心（ネタバレ注意）

4つの大きなミスリードがあります。これを理解した上でシナリオを扱ってください。

1. **主人公は声を失っている** — プレイヤーには最後まで明かされない。「伝えた」「応えた」という表現で誤認させる
2. **「キミノコエ」のダブルミーニング** — 「君の声」= 弟スグの声 / 「君の声」= 主人公自身の失われた声
3. **2人の白い服の少年** — 実在する少年（イロの友人の子供）と、スグの霊体（本物の弟）
4. **記憶の混濁** — ep_0 は歪んだ記憶（喧嘩したまま事故へ）。ep_0_β が真実（仲直りしていた）

------

## シナリオの記法ルール（重要）

### 主人公の声について

- **現在時点では声を出してはいけない**
- **過去のエピソード内では声を出してOK**
- **手話を知っている人との会話**では「伝えた」「応えた」と書く（ミスリード）
- **霊体を出すかどうか**は意図的に使い分ける
  - 霊体あり: ep_1, ep_2, ep_3
  - 霊体なし: ep_4, ep_5, ep_6, ep_7

### エピソード後の描写

- エピソード終了後は必ず「……。…………。………………。」で始まる
- 霊体ありエピソードの後に霊体描写を入れる
- 霊体なしエピソードは温かい余韻で終わる
